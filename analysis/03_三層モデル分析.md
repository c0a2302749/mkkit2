# 三層モデルの世界としての整合性

## 三層構造

```
 Governance Layer (src/governance/)
      │
      │ W (risk_score), warning
      ▼
 Participant Layer (src/agent/) ←──→ SNS Layer (src/social/)
 ┌─────────────────────┐   S_O,S_R   ┌─────────────────────┐
 │  AgentManager (LLM) │   timeline   │  SocialGraph        │
 │  OpinionDynamicsEng │ ◄────────── │  (有向グラフ)       │
 │  O, R 数理更新モデル │             │  TimelineManager    │
 │  α,β,γ,λ,b,w_sys   │ ──────────► │  (TL構築・拡散)     │
 └─────────────────────┘   actions   └─────────────────────┘
```

**三層 = Participant / SNS / Governance**

## 1. 情報が流れる — 成立

| 経路 | 仕組み | 実装箇所 |
|------|--------|---------|
| エージェント → SNS | PROPOSE/COMMENT/SUPPORT/OPPOSE/VOTE が `TimelineManager.add_post()` に蓄積 | `simulation.py:105` |
| SNS → エージェント(数理) | `compute_social_signals()` でフォロー先の平均意見O・平均リスク認識Rを計算 | `engine.py:11-27` |
| SNS → エージェント(LLM) | `get_timeline()` でフォロー先の投稿を自然言語プロンプトとして構成 | `timeline.py:64-84` |
| グラフ動的更新 | SUPPORT/OPPOSE時に提案者を自動フォロー (`update_network()`) | `graph.py:80-86` |

SNS層が双方向の情報伝達路として機能している。数理信号（S_O, S_R）と自然言語（タイムラインテキスト）の **2経路** で情報が伝播する。ネットワークトポロジも5種（random, small-world, scale-free, community, complete）から選択可能で、異なる拡散パターンを表現できる。

## 2. 意見が変わる — 成立

| 駆動要因 | 数式 | パラメータ |
|---------|------|-----------|
| 同調（社会的信号） | +α·S_O | α（同調性）0.15-0.7 |
| リスク回避 | -β·R | β（リスク回避）0.1-0.7 |
| 初期スタンス固執 | -γ·(O-O_0) | γ（信念抵抗）0.1-0.6 |
| リスク認識更新 | R(t) = (1-λ)·R(t-1) + λ·sigmoid(w_sys·W + w_sns·S_R + b) | λ, w_sys, w_sns, b |

リスク認識更新式は **システムリスクW**（Governance層由来）・**社会的信号S_R**（SNS層由来）・**個人バイアスb** の3要因で構成されており、三層すべてからの入力が意見変化に影響する因果連鎖ができている。

6ペルソナ（合理的主体/同調型/情報収集型/リスク過大評価型/懐疑型/扇動者）にα/β/γパラメータがマッピングされており、多様な意見変化パターンを表現可能。

## 3. 投票する — 成立

| 要素 | 仕様 |
|------|------|
| 提案生成 | PROPOSEアクションで Proposal 作成、全エージェントに可視 |
| 投票方法 | VOTEアクション — 全OPEN提案に同時投票可能（マルチ投票） |
| 解決ルール | 最終ターンに集計、賛成＞反対 → PASSED |
| フォールバック | LLMの自由回答から投票意図をregex抽出 (simulation.py:_extract_vote_from_rationale) |

提案→討論→投票という集団意思決定フローが一貫して実装されている。LLMの自由回答を構造化データにパースする2段構え（JSONパース → rationale抽出）も備えており、実運用に耐える。

## 4. Governance層の位置づけ — インターフェースは確立、実効ロジックは未実装

現在のGovernanceは `GovernanceStub`（常にW=0, warning=""）のみ。

- `GovernanceLayer(ABC)` — 抽象インターフェースは確立
- `compute_risk_score()` / `build_system_warning()` — Engineへの組み込み完了 (`simulation.py:59-64`)
- 実際のリスク評価ロジックは未実装（Phase D で実装予定）

評価: 「層としては存在するが現状はバイパス状態」。インターフェースと結合は完了しており、リスク評価ロジックを追加すれば即座に三層完全ループとなる。

## 5. ループの閉じ方

```
Governance ──W──► Participant ──actions──► SNS
   ▲                                        │
   │◄──────── S_O, S_R, TL ────────────────┘
   │
   └── (state情報でリスク評価) ──┘  ※スタブのため未発火
```

- **閉じている**: Participant ↔ SNS の相互ループ
- **閉じている**: Governance → Participant の一方向
- **閉じていない（設計対象外）**: Governance → SNS の直接経路（GovernanceはSNSの構造を操作しない設計）
- **未発火**: SNS → Governance のフィードバック（スタブのため）

## 結論：三層モデルは世界として成立しているか？

**成立している。**

| 要件 | 評価 | 根拠 |
|------|------|------|
| 情報が流れる | ✅ | 数理信号+自然言語の2経路、5トポロジ、動的フォロー |
| 意見が変わる | ✅ | 3要因（同調/リスク/固執）+6ペルソナ+層間因果連鎖 |
| 投票する | ✅ | 提案→討論→多数決、マルチ投票、フォールバック抽出 |
| 三層の因果連鎖 | ✅ | Governance→Participant→SNS がEngineに実装済み |
| Governanceの実効性 | ⚠️ 未実装 | スタブのみ。インターフェースと結合は完了 |

**現状の実質的な動作は「二層＋Governanceインターフェース」**。Governance層に実際のリスク評価ロジック（ex: 意見分散の閾値監視, 誤情報ラベル, エコーチェンバー検知）が入れば完全な三層ループとなる。

このモデル上で「三層だけで」動かす限り、最低限のシミュレーション（情報が流れる・意見が変わる・投票する）は **実装上の欠陥なく成立しており、世界としての自己無撞着性は担保されている**。
