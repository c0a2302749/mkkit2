## 4. Phase 4: 参加者層（LLM行動決定）

### LLMProvider(ABC)

```python
class LLMProvider(ABC):
    @abstractmethod
    async def invoke(self, prompt: str) -> str: ...
```

| 実装 | 用途 |
|------|------|
| `AzureProvider` | Azure OpenAI（唯一の実装） |

### AgentManager

- `decide_action(agent, timeline, warning) → Action`
- プロンプト構成: 現在の意見・リスク認識・タイムライン情報・警告文 → 行動選択 + rationale
- 行動: 提案・コメント・支持・反対・投票・何もしない

### LLM採用の論証（3段構成）

**第1段（数理モデルの限界）**: FJモデル等の既存数理モデルは意見変化の方向と量は記述できるが、「なぜ変化したか」——警告ラベルへの反応か、多数派への同調か——というメカニズムの区別を出力できない。

**第2段（要件の定義）**: 制度的介入の効果をメカニズムとして評価するには、エージェントが「どの情報に基づいて判断を変えたか」という理由（rationale）を観測できる必要がある。

**第3段（LLMによる充足）**: LLMエージェントは rationale をテキストとして生成するため、この要件を満たす。数理モデルは O,R の計算、LLM は rationale 生成と行動選択という役割分担により、両者を併用するハイブリッド設計とする。

**完了条件**: `AgentManager.decide_action()` がエラーなく呼び出せることを確認（実際のAzure呼び出しは結合テストで検証）

## 6. Phase 6: 評価指標モジュール

### 統計ユーティリティ

- `Statistics.compute_opinion_distribution(agents)`

### エコーチェンバー

- **主指標**: クラスタ内意見分散 / クラスタ間意見距離 の比率
- **補助**: assortativity, modularity の推移

### 同調

- **主指標**: 意見変化方向がタイムライン平均意見の方向と一致する割合
- **補助**: 同調項 `α_i · S_i^O(t)` の寄与度時系列

### 情報カスケード（ミクロ）

- **私的情報の無視率**: 初期意見 `O_i(0)` と TL平均 `S_i^O(t)` が対立する状況で、LLMが自身の初期スタンスに反して追従した割合

### 情報カスケード（マクロ）

- **経路依存性（ラン間分散）**: 初期意見分布とネットワーク構造を固定し、乱数シード（発言順序）のみ変更したマルチランで最終平均意見の分散を測定

**完了条件**: 各指標関数が `pytest` で正しく計算されることを確認（テストデータで期待値を検証）

## 7. Phase 7: ログ・出力基盤

### OutputManager

- `OutputManager(base_dir)` — タイムスタンプベースのrun directoryを作成
- `record_turn(turn, agents, turn_actions, social_signals, config)` — ターンデータを蓄積
- `save_all(config)` — 以下の4ファイルを出力

### 出力ファイル

| ファイル | 形式 | 優先度 | 内容 |
|---------|------|--------|------|
| `actions.json` | JSON | **最高** | ターン×エージェントの行動 + rationale（LLM採用根拠となる中核データ） |
| `opinion_trajectory.csv` | CSV | 高 | 全ターン×全エージェントの O,R,S_O,S_R 推移。プロット・分析用 |
| `statistics.csv` | CSV | 中 | ターン単位の平均 O, R, 標準偏差。Phase C の指標計算用 |
| `summary.json` | JSON | 中 | 実行設定・最終集計・n_actions 等のメタデータ |

### CLI引数

- `--log-dir` : 出力先ディレクトリ（デフォルト: `logs/`）

**完了条件**: `uv run main.py --null --turns 2 --agents 4` で上記4ファイルが出力される
