# 研究計画

## 研究の1行目的

**三層シミュレーション基盤（参加者層/SNS層/ガバナンス層）を実装し、帰無モデル比較によりSNS層に起因する意見動態現象（エコーチェンバー・同調・情報カスケード）の生起を統計的に検証する。**

---

## フェーズ構成

| Phase | 名称 | 依存 | 位置づけ |
|-------|------|------|---------|
| A | ペルソナ-パラメータ連動検証 | 実装Phase 1-4完了後 | 基盤動作の最小確認 |
| B | SNS層動作検証 | 実装Phase 3完了後 | ネットワーク拡散の定性的妥当性確認 |
| **C** | **帰無モデル比較検証** | **全層実装後** | **研究の中核** |
| D | ガバナンス結合確認 | 実装Phase 5完了後 | 拡張性の確認 |

## Phase A: ペルソナ-パラメータ連動検証

### 目的
α/β/γパラメータがLLMエージェントの行動に期待通り反映されているかを確認する。パラメータ空間から行動多様性へのマッピングが正しいことの保証。

### 方法
- **実験条件**: 6ペルソナ × 2条件（SNS層有/無）
- **試行回数**: 各条件20 trial（乱数シードを変えて実行）
- **LLM**: Azure OpenAI
- **エージェント数**: 8
- **ラウンド数**: 5

### 評価指標

| ペルソナ | 期待される行動傾向 | 確認方法 |
|---------|------------------|---------|
| 合理的主体（高γ） | 意見を変えにくい | 意見変化量の平均 |
| 同調型（高α） | TL多数派に追従 | 変化方向一致率 |
| 情報収集型（中αβγ） | 様子見傾向 | DO_NOTHING選択率 |
| リスク過大評価型（高β） | 警告に敏感 | リスク認識値の変化 |
| 懐疑型（低α, 高γ） | 初期スタンス固執 | 意見不変率 |
| 扇動者（低β, 高γ） | 活発に発信 | 発言頻度（P+/C+/S+） |

### 統計基準
- 行動分布のペルソナ間差: χ²検定（p<0.05）
- 効果量: Cramér's V（参考値として報告）

### 成功条件
6ペルソナ中4以上で期待傾向との有意な一致が確認される。

### 1行要約
> AzureLLMがα/β/γパラメータの期待行動傾向を再現するか、6ペルソナ×20trialの行動分布で検定する。

## Phase B: SNS層動作検証

### 目的
ネットワークトポロジに依存した情報拡散が正しく再現されていることを確認する。LLMオフ（数理モデルのみ）で、拡散パターンが既存ネットワーク知見と定性一致するか検証。

### 方法
- **実験条件**: 5トポロジ × 3乱数シード
- **LLM**: オフ（行動決定はランダム選択に固定）
- **エージェント数**: 8
- **ラウンド数**: 20

### 評価指標
- 情報到達率の時系列（最初のターンから何エージェントに届いたか）
- クラスタ係数の推移
- 平均意見のトポロジ間比較

### 既存知見との定性一致基準

| トポロジ | 期待される特性 | 確認観点 |
|---------|--------------|---------|
| スモールワールド | 高クラスタ係数 + 短平均距離 | 高速拡散＋局所凝集 |
| スケールフリー | ハブ経由の伝搬 | 少数ノードが大多数に到達 |
| ランダム | 低クラスタ、一様拡散 | トポロジ間比較のベースライン |
| コミュニティ | コミュニティ内偏在 | コミュニティ間の到達率差 |
| 完全グラフ | 全エージェント即時到達 | 1ターン目で全到達 |

### 統計基準
- 記述統計のみ（平均 ± SD）
- 既存知見との定性的一致を記述

### 成功条件
5トポロジ中3以上で期待される拡散特性との定性一致が確認される。

### 1行要約
> 5トポロジの拡散特性を記述統計で比較し、スモールワールド・スケールフリーの既知特性との定性一致を確認する。

## Phase C: 帰無モデル比較検証（研究の中核）

### 目的
SNS層（社会的信号 + 情報伝播）が、エコーチェンバー・同調・情報カスケードの原因であることを統計的に示す。

### 方法

#### 実験群 vs 帰無群の定義

| 要素 | 実験群 | 帰無群 |
|------|--------|--------|
| `S_i^O(t)` | フォロー先の平均意見 | `0` |
| `S_i^R(t)` | フォロー先の平均リスク認識 | `0` |
| タイムライン | フォロー先の投稿一覧 | `[]` |
| ネットワーク | SocialGraph 構築 | 同数の孤立ノード |
| Governance | GovernanceStub（W=0, warning=""） | GovernanceStub（W=0, warning=""） |

#### 帰無群の完全な退化更新式

GovernanceStub（W=0）により：

```
R̃_i(t) = w_sys · 0 + w_sns · 0 + b_i = b_i
R_i(t) = (1 - λ_i) · R_i(t-1) + λ_i · sigmoid(b_i)
O_i(t+1) = clip(O_i(t) - β_i · R_i(t) - γ_i · (O_i(t) - O_i(0)), 0, 1)
```

エージェントは自身のバイアス `b_i` にのみ基づいてリスク認識を更新し、同調項は発火しない。意見変化は「リスク回避による減衰」と「初期スタンス固執による復元」のみで駆動される。

#### なぜ`S_i^O=0, TL空, GovernanceStub`なのか

| 選択肢 | 問題点 |
|--------|--------|
| α項だけ0にする | 他のSNS経路（TLのLLM入力, S_R）が残る → 「SNS層なし」ではない |
| Governanceは生かす | W≠0になりうる → SNS層の効果とGovernanceの効果が交絡 |
| ネットワークを削除 | コードパスが変わる → 「原因の差」以外に「コードの差」が混入 |

帰無群でもコードパスは同一に保ち、入力値（社会的信号・タイムライン）のみを0/空にすることで、「SNS層からの情報入力の有無」に原因を限定する。

### 実験設定

| パラメータ | 値 |
|-----------|-----|
| エージェント数 | 8 |
| ラウンド数 | 10 |
| 乱数シード | 50（実験群・帰無群で同一シードのペア実行） |
| トポロジ | ランダム（実験群のみ） |
| LLM | Azure OpenAI |
| Governance | GovernanceStub（実験群・帰無群とも） |

### 評価指標と統計基準

| 現象 | 指標 | 操作定義 | 統計検定 | 有意水準 |
|------|------|---------|---------|---------|
| エコーチェンバー | クラスタ内分散/クラスタ間距離比率 | 時間経過に伴いクラスタ内の意見が収束し、クラスタ間の距離が拡大する度合い | Welchのt検定 | p<0.05 |
| 同調 | 意見変化一致率 | エージェントの意見変化方向がTL平均の方向と一致する割合 | Mann-Whitney U | p<0.05 |
| カスケード（ミクロ） | 私的情報無視率 | 初期意見とTL平均が対立する状況で、初期スタンスに反して追従した割合 | 二群比率検定 | p<0.05 |
| カスケード（マクロ） | ラン間分散（経路依存性） | 発言順序（乱数シード）のみを変えたマルチランにおける最終平均意見の分散 | F検定（等分散検定） | p<0.05 |

### 報告の方針

- 各指標の平均差 ± SD を報告
- **効果量は全て参考値として報告**（Cohen's d, r, リスク比など）
- 中間発表では事前に有意水準を固定せず、今回は探索的位置づけであることを明記
- 本実験で得られた効果量を元に、次フェースでは事前登録した閾値を設定する

### 成功条件

4指標中3指標以上で有意差（p<0.05）が確認されること。

### 1行要約
> 実験群50runs vs 帰無群50runsの4指標を比較し、SNS層に起因する意見動態現象の生起を探索的に検証する（p<0.05, 効果量は参考値）。

## Phase D: ガバナンス結合確認

### 目的
`GovernanceLayer(ABC)` を介した層間データフローが正常に機能することを確認する。スタブ実装の差し替えでEngineがエラーなく稼働することを検証。

### 方法
- **実験条件**: GovernanceStub（常にW=0） vs 警告付与スタブ（ランダムに警告文字列を生成）
- **エージェント数**: 8
- **ラウンド数**: 3
- **乱数シード**: 5
- **LLM**: AzureProvider（小ターン数で実施）

### 確認項目
1. 全ターン例外なく完了するか
2. `risk_score` が `OpinionDynamicsEngine` に正しく渡されるか
3. `warning` が `AgentManager.decide_action()` に正しく渡されるか
4. スタブ差し替え前後でEngineの動作が変わらないか

### 成功条件
全5シード × 3ラウンドで例外なく完了し、スタブの出力が下流の層に正しく伝達されていること。

### 1行要約
> GovernanceStub と 仮の警告実装をEngineに挿入し、層間データフローが例外なく循環することを確認する（LLM呼び出しは実際にAzureを通す）。

## 付録: Phase C 成功に至らなかった場合の対応

| ケース | 原因仮説 | 対応 |
|-------|---------|------|
| 同調だけ有意でない | エージェント数8が少なすぎて統計的検出力不足 | n=32で再試行、または指標の定義を見直し |
| 全指標で有意差なし | SNS層の効果が微弱か、実験群と帰無群の差が小さすぎる | 効果量を確認し、d<0.2なら設計自体の再検討（パラメータ空間の再設計） |
| 帰無群でも「同調」が観測 | LLMのランダム性によるアーティファクト | 同一seed・同一プロンプトで再現確認、再現するなら指標の操作定義が不適切 |
| エコーチェンバーのみ有意 | ネットワーク構造の効果は強いが、LLM意思決定への影響が小さい | TL情報量の調整、プロンプト設計の見直し |

## 補足: R_i(t) の更新に関する実装注意

帰無群でも `R_i(t)` の更新は行われる。`w_sns=0` により社会的信号の影響はゼロになるが、自身のバイアス `b_i` に基づくリスク認識は継続する。これにより帰無群でもエージェントは「個人のリスク認識」を持ち、それが意見変化（β項）に影響を与える。

実験群との比較は「社会的信号の有無」に限定され、「リスク認識の有無」ではない。この点は論文でも明記すること。
